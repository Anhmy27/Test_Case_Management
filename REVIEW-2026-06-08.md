## 🔴 Critical — must fix before any deployment

1. **Real secrets committed to git** (`backend/.env`). Tracked file contains a live Jira
   password, `JWT_SECRET`, and admin password. `.gitignore` has `.env/` (trailing slash →
   ignores a *directory*, not the file), so it never worked.
   - Rotate the Jira password and JWT secret now (they are in git history).
   - `git rm --cached backend/.env`, fix `.gitignore` to `.env`, add a `.env.example`.
   - Treat history as compromised; consider `git filter-repo` to purge.

2. **Weak/default JWT secret + no revocation** — **fixed 2026-06-10**.
   No fallback secret; startup fails if `JWT_SECRET` missing; weak secrets blocked in production.
   Default TTL is `8h`; JWT embeds `tokenVersion` and admin password/role/deactivate/delete bumps version to revoke existing sessions.

3. **JWT stored in `localStorage`** — **fixed 2026-06-09**.
   Session JWT moved to httpOnly cookie; CSRF header/cookie protection added for mutating API calls; `helmet` enabled.

4. **Open self-registration** — **accepted 2026-06-10**.
   `POST /api/auth/register` stays open for internal employee onboarding, similar to self-hosted Jira access patterns.

5. **No rate limiting on auth routes** — **fixed 2026-06-11** (login/register only).
   MongoDB-backed rate limits in `AuthRateLimit` collection (`authRateLimitService.js`, `authRateLimitMiddleware.js`).
   - **Login** counts every attempt (failed or successful) per IP + per email.
   - **Register** counts only **successful account creation**; duplicate-email / validation failures do not consume quota; quota is reserved atomically before `User.create` and refunded on unexpected create failure.
   - **Jira `POST /api/jira/log-bug` is not rate-limited** (by design).
   - Env vars use distinct naming so behaviour is obvious:
     - Login: `AUTH_LOGIN_ATTEMPT_LIMIT_MAX_PER_IP`, `AUTH_LOGIN_ATTEMPT_LIMIT_MAX_PER_EMAIL`, `AUTH_LOGIN_ATTEMPT_LIMIT_WINDOW_MINUTES`
     - Register: `AUTH_REGISTER_SUCCESS_LIMIT_MAX_PER_IP`, `AUTH_REGISTER_SUCCESS_LIMIT_MAX_PER_EMAIL`, `AUTH_REGISTER_SUCCESS_LIMIT_WINDOW_MINUTES`
   - Defaults: login 100 attempts/IP + 10 attempts/email per 15 min; register 20 successful accounts/IP + 1 successful account/email per 60 min.
   - Optional `TRUST_PROXY=1` for correct client IP behind reverse proxy.

6. **Regex injection / ReDoS in search** (`utils/versioning.js`) — **fixed 2026-06-09**.
   Input is escaped before `$regex` is built.
