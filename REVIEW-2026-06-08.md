# Senior Code Review — Test Case Management (2026-06-08)

Fresh review against the **current** code. The earlier `REVIEW.md` is partly stale —
several issues it flagged are now fixed (Jira routes are authenticated, `jiraController`
has a single export, Excel import has size/MIME checks, the automation-results route now
sits behind auth).

## Overall assessment

For a fresher + AI effort the structure is genuinely good: clean route → controller →
service layering, a soft-delete + versioning abstraction, a reasonable React component
split, and sensible Mongoose models. The gaps are the expected ones — **security
hardening, durability, validation, and testing** — none of which are production-ready yet.

---

## 🔴 Critical — must fix before any deployment

1. **Real secrets committed to git** (`backend/.env`). Tracked file contains a live Jira
   password, `JWT_SECRET`, and admin password. `.gitignore` has `.env/` (trailing slash →
   ignores a *directory*, not the file), so it never worked.
   - Rotate the Jira password and JWT secret now (they are in git history).
   - `git rm --cached backend/.env`, fix `.gitignore` to `.env`, add a `.env.example`.
   - Treat history as compromised; consider `git filter-repo` to purge.

2. **Weak/default JWT secret + no revocation** (`authMiddleware.js:6` falls back to
   `'replace-me-secret'`; `.env` ships `super-secret-change-me`). Tokens are 7-day,
   stateless, and cannot be revoked — a deactivated/compromised user keeps access for a
   week. Fail-fast if `JWT_SECRET` is unset; shorten lifetime; add refresh/revocation.

3. **JWT stored in `localStorage`** (`WorkspaceShell.tsx`, `app/page.tsx`). Any XSS = full
   account takeover. Move to httpOnly cookies (+ CSRF) or add a strict CSP. No `helmet`
   today.

4. **Open self-registration** (`POST /api/auth/register`). Anyone on the network can create
   an `employee` account — no verification/invite/approval. Remove or gate it.

5. **No rate limiting anywhere.** Login, register, and the Jira proxy are brute-forceable.
   Add `express-rate-limit` on `/api/auth/*` at minimum.

6. **Regex injection / ReDoS in search** (`utils/versioning.js:31`). User `search` goes
   straight into `$regex` unescaped — a crafted pattern can hang the event loop. Escape the
   input or use a text index.

---

## 🟠 High-priority — architecture & robustness

7. **Automation job runner is in-memory and non-durable** (`automationJobRunner.js`).
   `activeRunIds` is a module-level `Set`; jobs run via `setImmediate`. A restart mid-run
   leaves `TestRun.status = 'running'` forever (no reaper); won't work with >1 instance; no
   real queue/retry/concurrency cap. Use a persistent queue (BullMQ/Redis) + a startup
   reconciler for orphaned runs.

8. **`testManagementService.js` is a 2,751-line god file.** Biggest maintainability risk.
   Split by domain (projects / versions / test-cases / test-plans / test-runs).

9. **No input validation layer.** Controllers pass `req.body`/`req.query` straight into
   services. Add schema validation (Zod/Joi) at the route boundary.

10. **Error stack traces leak by default** (`errorMiddleware.js`). `NODE_ENV` is never set,
    so prod will leak stacks and raw upstream Jira HTML error bodies. Set
    `NODE_ENV=production` and sanitize upstream error bodies.

11. **Local-disk artifact storage** (`artifactStore.js`, failure screenshots). Breaks with
    multiple replicas / ephemeral containers; no retention/cleanup. Use object storage.

12. **Fragile Jira integration** (`jiraService.js`). Scrapes login HTML for `atl_token`,
    posts to internal `*.jspa` endpoints with hard-coded custom field IDs, single shared
    account, re-auths on every call. Prefer the official REST API + scoped token + session
    caching.

13. **SSRF surface via automation** (`runOrchestrator.js` / dry-run). Admin-supplied
    `baseUrl`/`goto` drive a server-side headless browser to arbitrary URLs; the `upload`
    action (`playwrightExecutor.js:280`) resolves arbitrary server filesystem paths. Add a
    `baseUrl` allowlist and disallow absolute server paths.

## 🟡 Bugs & smaller issues

- **Default admin seeded from `.env`** (`seedAdmin.js`) with `admin@example.com` /
  `12345678`. Don't ship defaults; require explicit opt-in.
- **`applyAutomationResults` has no role authorization** — authenticated only + an
  `x-automation-secret` header check. Confirm the secret check rejects when missing.
- **Bleeding-edge deps**: Express 5, Mongoose 9, Next 16, React 19. Pin and verify for prod.
- **No DB index review** beyond versioning defaults; `$regex` searches table-scan.

---

## 🟢 Missing for production use

- **Tests: zero** (`"test": "echo no tests"`). Cover auth, versioning, automation runner.
- **CI/CD**: no pipeline, no lint/typecheck gate, no app Dockerfile (compose only runs Mongo).
- **Observability**: no structured logging, request logging, metrics, or error tracking.
- **Health/readiness endpoint** for orchestration (only `/` banner).
- **Account lifecycle**: no password reset, email verification, complexity policy, lockout.
- **Audit log** of who changed/ran what.
- **Backups & retention** for Mongo and artifacts.
- **HTTPS/TLS + reverse proxy** guidance; **graceful shutdown** (drain runs, close Mongo).
